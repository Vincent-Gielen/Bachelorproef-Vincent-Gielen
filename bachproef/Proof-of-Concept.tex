\section{PoC 1: Gebruik maken van  een API}
In deze PoC wordt er getracht om via het mainframe contact te leggen met een API. Er wordt contact gelegd met een private API die op hetzelfde mainframe draait, alsook de publieke API van \href{https://github.com/open-meteo/open-meteo}{open-meteo} om het weer in Madrid te verkrijgen. Zo kan er ook gekeken worden of er een verschil is als een API op een mainframe draait of niet, en welke parameters daarvoor moeten worden veranderd.

\subsection{Python}
In de code van de Python bestanden \ref{mint:privateAPIPython} en \ref{mint:publiekeAPIPython} is het proces bijna hetzelfde. De requests package van Python wordt geïmporteerd. Deze veelgebruikte package zorgt ervoor dat API requests op een simpele manier kunnen worden uitgevoerd. Voor de private API moet er een gebruikers-ID en een wachtwoord worden meegegeven om juist te kunnen aanmelden op de het mainframe. Hiervoor wordt een extra package van Python gebruikt. Voor de publieke API moeten er bepaalde parameters worden meegegeven om de informatie te verkrijgen over een specifieke locatie (hier Madrid). Bij het gebruik van de requests package wordt automatisch geverifieerd of de server (het systeem dat wordt aangesproken) wel veilig is. Dit wordt gedaan aan de hand van internationale standaarden en certificaten. Bij de private API is dit manueel uitgezet, omdat we het systeem kennen, en de certificaten dit voorbeeld onnodig complex zouden maken. Nadat de request al dan niet succesvol is verstuurd, wordt er een response ontvangen. Als dit antwoord goed is (met de code 200), dan worden de resultaten afgebeeld op het scherm.

\begin{listing} [h]
    \begin{minted}[frame=lines, framesep=2mm, linenos]{python}
        # De Python package requests en getpass worden geimporteerd
        import requests
        from getpass import getpass
        
        # De credentials om in te loggen worden aan de gebruiker gevraagd en opgeslaan
        myCreds = ('','')
        
        if myCreds[0] == '':
            userid = input("Give your userid: ")
            myCreds = (userid,'')
        
        if myCreds[1] == '':
            passwd = getpass("Give your password: ")
            myCreds = (myCreds[0], passwd)
        
        # De URL naar de private API van mainframeyin
        url = "https://mainframeyin:8092/LWWAPI/API/API"
        
        # De credentials worden meegegeven zodat correct kan worden aangemeld
        # Er wordt niet geverifieerd of deze API veilig is, omdat we deze zelf beheren
        response = requests.get(url, auth=myCreds, verify=False)
        
        if (response.status_code != 200):
            raise Exception(f"{response.status_code} - {response.text}")
        else:
            # We verkrijgen de gegevens in JSON formaat
            ApiList = response.json()
        
        # De verschillende API namen worden afgebeeld
        for api in ApiList['API']:
            print(api['APIName'])
    \end{minted}
  \caption[Private API op hetzelfde mainframe in Python]{\label{mint:privateAPIPython}Contact leggen met een private API op hetzelfde mainframe met Python}
\end{listing}

\begin{listing} [h]
    \begin{minted}[frame=lines, framesep=2mm, linenos]{python}
        # De Python package requests wordt geimporteerd
        import requests
        
        # De coördinaten voor Madrid
        params = {
            "latitude": 40.4168,
            "longitude": -3.7038,
            "current_weather": "true",
            "temperature_unit": "celsius"
        }
        
        # De URL naar de publieke API van open-meteo
        url = "https://api.open-meteo.com/v1/forecast"
        
        # Er zijn hier geen credentials nodig, aangezien de API public beschikbaar is
        # We geven de parameters mee, zodat we het weer van Madrid krijgen
        response = requests.get(url, params=params)
        
        # Als er een error is, wordt die hier getoond
        if response.status_code != 200:
            raise Exception(f"{response.status_code} - {response.text}")
        else:
            # We verkrijgen de gegevens in JSON formaat
            weather_data = response.json()
        
        # We halen enkele gegevens uit de JSON
        current = weather_data['current_weather']
        temp = current['temperature']
        wind = current['windspeed']
        
        # Het resultaat wordt afgebeeld
        print(f"Het weer in Madrid:")
        print(f"Temperatuur: {temp} graden Celsius")
        print(f"Windsnelheid: {wind} km/h")
    \end{minted}

\subsection{Rexx}
In de code van de Rexx bestanden \ref{mint:privateAPIRexx} en \ref{mint:publiekeAPIRexx} wordt gebruik gemaakt van het Unix commando cURL (Client URL). Dit commando wordt veel gebruikt voor het communiceren tussen systemen en is dus niet specifiek voor Rexx. Oorspronkelijk zou er voor Rexx gebruik gemaakt worden van de z/OS Client WET (Web Enablement Toolkit), maar dit bleek te complex voor het relatief kleine voorbeeld. Ervaren systeemprogrammeur Lionel Dyck gaf dit advies:

\blockquote[Lionel B. Dyck - IBM Z Champion on Discord]{I once looked at WET and, tbh(to be honest), gave up - seemed too complex for what it does.  Using bpxwunix with standard unix tools seemed easier/simpler and more maintainable down the road.  Just my \$0.01 - ymmv(your mileage may vary)}

Het algemene proces in Python en Rexx is hetzelfde. Er wordt een request gestuurd naar een API via een url, met enkele parameters of credentials. In de Rexx code moeten we de parameters en credentials op handmatige manier toevoegen aan de url voordat we deze kunnen versturen met het cURL commando. Het cURL commando kan op verschillende manieren worden aangepast aan de noden van de use case, gelijkaardig aan de requests package van Python. We geven ook een stem variabele mee, dit werkt op een gelijkaardige manier als een array, waar de response zal in terechtkomen. Nadat we checken op errors (rc = 0), kunnen we in de response met handige string manipulatie de gewenste resultaten afbeelden.

\begin{listing} [h]
    \begin{minted}[frame=lines, framesep=2mm, linenos]{python}
        /* REXX */
        
        /* De URL naar de private API */
        url = 'https://mainframeyin:8092/LWWAPI/API/API'
        
        /* De credentials om in te loggen aan de gebruiker vragen */
        /* We gebruiken PARSE PULL zodat hoofdletters behouden blijven */
        say "Give your userid: "
        parse pull userid
        myCreds.1 = userid
        
        say "Give your password: "
        parse pull passwd
        myCreds.2 = passwd
        
        /* We bouwen het curl commando op:
        -k : --insecure (vergelijkbaar met verify=False in Python)
        -u : --user (voor Basic Authentication)
        */
        commando = "curl -k -u "myCreds.1":"myCreds.2" '"url"'"
        
        /* Voer het Unix commando uit */
        rc = bpxwunix(commando,,response.,error.)
        
        /* Als er een error is, wordt die hier getoond */
        if rc <> 0 then do
            say "Error executing curl. Return code:" rc
            do i = 1 to error.0
                say error.i
            end
            exit 8
        end
        
        /* Verwerking van de JSON response */
        /* We zoeken naar de APINames in de JSON */
        ApiList = response.1
        
        say "Lijst met API's:"
        do while pos('"APIName":"', ApiList) > 0
            /* De verschillende API namen worden afgebeeld */
            parse var ApiList '"APIName":"' name '"' ApiList
            say name
        end
        
        exit 0
        
    \end{minted}
    \caption[Private API op hetzelfde mainframe in Rexx]{\label{mint:privateAPIRexx}Contact leggen met een private API op hetzelfde mainframe met Rexx}
\end{listing}

\begin{listing} [h]
    \begin{minted}[frame=lines, framesep=2mm, linenos]{python}
        /* REXX */
        
        /* We moeten de parameters in de url steken van de publieke API */
        url = 'https://api.open-meteo.com/v1/forecast?latitude=40.4168' || ,
        '&longitude=-3.7038&current_weather=true&temperature_unit=celsius'
        
        /* We maken het Unix commando met de publieke API van open-meteo*/
        commando = "curl '"url"'"
        
        /* Dit is een Rexx commando (bpxwunix) om een Unix commando (curl)
        uit te voeren. Het antwoord komt in response. terecht, enige errors in error.*/
        rc = bpxwunix(commando,,response.,error.)
        
        /* Als er een error is, wordt die hier getoond */
        if rc <> 0 then do
            say "Error executing curl. Return code:" rc
            do i = 1 to error.0
                say error.i
            end
            exit 8
        end
        
        /* We zoeken specifiek binnen de 'current_weather' sectie
        We snijden de tekst van links naar rechts
        op zoek naar temperature en windspeed */
        parse var response.1 'current_weather":{' data '}' .
        parse var data 'temperature":' temp ','
        parse var data 'windspeed":' wind ','
        
        say "Het weer in Madrid:"
        say "Temperatuur: " temp "graden Celsius"
        say "Windsnelheid:" wind "km/h"
        
        exit 0
        
    \end{minted}

\begin{itemize}
\item \textbf{Integratie met tools op het mainframe:}
Rexx vertoont een zeer natuurlijke integratie met het mainframe-ecosysteem door het gebruik van \texttt{bpxwunix}, waarmee direct gebruik kan worden gemaakt van de onderliggende Unix System Services (USS). Python op het mainframe vereist een specifiekere inrichting, zoals een correct geconfigureerde \texttt{venv} (virtual environment) en de installatie van poorten van specifieke libraries, maar biedt vervolgens een abstractielaag die de complexiteit van mainframe-specifieke protocollen verbergt.
\item \textbf{Integratie met tools buiten het mainframe:}
Python is hier de duidelijke winnaar dankzij de \texttt{requests} package, die specifiek is ontworpen voor moderne webcommunicatie en naadloos omgaat met zaken als SSL-verificatie en JSON-parsing. Rexx moet voor externe integratie vertrouwen op externe tools zoals \texttt{curl}. Hoewel dit effectief is, vereist het meer handmatige configuratie van headers en parameters, wat de integratie met complexe externe API's bemoeilijkt.

\item \textbf{Herbruikbaarheid van de code:}
De Python-code is aanzienlijk herbruikbaarder door het gebruik van gestandaardiseerde packages en de mogelijkheid om logica eenvoudig te modulairiseren. In Rexx is de code vaak sterk afhankelijk van handmatige string-manipulatie (\texttt{parse var}) om data uit een response te extraheren. Indien de structuur van de API-output wijzigt, moet de Rexx-code op meerdere plekken worden aangepast, terwijl Python's JSON-objecten flexibeler zijn.

\item \textbf{Security vulnerabilities:}
Er is een aanzienlijk beveiligingsrisico bij de Rexx-implementatie omdat credentials als platte tekst worden meegegeven in het \texttt{curl}-commando. Dit kan leiden tot lekken in systeemlogs (zoals SDSF). Python biedt veiligere alternatieven zoals de \texttt{getpass} module en een meer volwassen afhandeling van HTTPS-certificaten, hoewel in deze PoC de verificatie voor de private API handmatig is uitgeschakeld voor testdoeleinden.

\item \textbf{Robuustheid van de code:}
Python biedt superieure foutafhandeling via exception handling (\texttt{try-except} structuren), waardoor de applicatie gecontroleerd kan reageren op netwerkfouten of ongeldige API-responses. Rexx vertrouwt op return codes (\texttt{rc}) en het handmatig uitlezen van de \texttt{error.} stem variabele. Dit maakt de code gevoeliger voor runtime errors, zeker bij het verwerken van grote hoeveelheden data waar de z/OS Client WET (Web Enablement Toolkit) tegen zijn limieten aanloopt.

\item \textbf{Ondersteuning van de taal:} 
Rexx is een legacy-taal die diep geworteld is in de mainframe-cultuur, maar de ondersteuning voor moderne webstandaarden is beperkt, wat blijkt uit het advies van experts om uit te wijken naar Unix-tools. Python daarentegen geniet van een enorme wereldwijde community en wordt door IBM actief gepusht als de moderne standaard voor mainframe-automatisering, wat resulteert in betere documentatie en modernere libraries.
