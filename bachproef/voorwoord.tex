Het idee voor deze bachelorproef is er gekomen door een combinatie van verschillende zaken. Ten eerste omdat ik de scripting taal Rexx aangenaam vond om mee te werken, maar ook door mijn interesse in automatisatie, en de grote focus in de mainframe industrie op modernisatie. Een combinatie van deze zaken bracht mij op het idee om de modernisatie golf naar Python te testen, en zien of Python echt een vervanger kan zijn voor Rexx. Ik ben verre van expert in Python, wat mij positief leek, omdat ik op die manier geen echte voorkeur heb voor een van beide.

Het originele idee van deze bachelorproef was een algemene vergelijking tussen Rexx en Python. Om specifieker te gaan heb ik aan de mainframe community op Discord ideeën gevraagd. Een van die ideeën was van Arthur Coucke. Hij stelde voor om Rexx en Python te vergelijken voor het genereren van JCL. Uit dit initiële idee vloeiden enkele interessante voorstellen om het hele automatisatie landschap te analyseren. Uiteindelijk heb ik besloten om het kleiner te houden, en de vergelijking tussen Rexx en Python te houden, maar Arthur's ideeën hebben wel een impact gehad in de vorming van deze bachelorproef. 

Daarnaast heb ik kunnen rekenen op de feedback van mijn promotor, Leendert Blondeel, om steeds kritisch te blijven kijken naar mijn idee en werk, zodat ik zeker op het juiste spoor bleef tijdens de uitwerking van de bachelorproef.

Om bepaalde ideeën voor de PoC vorming te geven heb ik heel veel steun gekregen van mijn co-promotor, Bobby Tjassens. Niet alleen inhoudelijk, maar ook voor de code kon ik doorheen heel het proces op de ervaring van Bobby rekenen. Wekelijkse meetings met toegang tot een systeem om op te werken, en ontelbaar veel code voorbeelden om mij op gang te helpen. Deze bachelorproef was zeker niet mogelijk geweest zonder Bobby.

Een groep die ik zeker niet mag vergeten bedanken, is de mainframe community, zowel op Discord als op Linkedin. Ik heb enorm veel input en feedback gekregen op elke vraag die ik heb gesteld. Ook het aantal antwoorden op mijn enquête heeft me enorm vooruit geholpen om de beste requirements en use cases te bepalen.
