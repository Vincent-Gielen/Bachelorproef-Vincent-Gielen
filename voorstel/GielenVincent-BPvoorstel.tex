\begin{abstract}
  Mainframe is een oude technologie die nog steeds intensief wordt gebruikt. In mainframe werd tot nu toe voornamelijk geautomatiseerd met Rexx. Echter stuit de arbeidsmarkt momenteel op een gebrek aan werkkrachten met voldoende kennis van Rexx. Vanuit die problematiek wil dit onderzoek de toekomst van de automatisatiestack op z/OS (het voornaamste besturingssysteem op mainframe) onderzoeken. Python wordt momenteel overwogen, en soms al gebruikt, als vervanger voor Rexx op mainframe. Python is in tegenstelling tot Rexx meer algemeen gekend. \textcolor{red} {Dus, gezien de modernisatiegolf die momenteel aan de gang is, en het gebrek aan kennis van Rexx bij velen op de arbeidsmarkt, lijkt Python de uitgelezen kandidaat. Maar is dat wel zo?} Dit onderzoek tracht volgende onderzoeksvraag te beantwoorden: Hoe ziet de automatisatie stack van de toekomst eruit op z/OS en wat staat er in de weg om van Python de nieuwe standaard te maken?
  
  \qquad Om de onderzoeksvraag te beantwoorden wordt eerst een literatuurstudie uitgevoerd. Op basis daarvan volgt een algemene vergelijkende studie van Rexx en Python, waar wordt getracht om enkele veel gebruikte Rexx commando’s te vervangen door Python. Tot slot wordt er met behulp van een PoC dieper ingegaan op één specifieke use case van Rexx, namelijk de File Tailoring Services voor het genereren van Job Control Language (JCL).
  
  \qquad Vermoedelijk wijzen de literatuurstudie en de vergelijkende studie uit dat, over het algemeen Python enorm veel mogelijkheden biedt, maar dat dat vaak gepaard gaat met verlies aan efficiëntie. Uit de PoC zal echter waarschijnlijk blijken dat in bepaalde gevallen, waaronder het genereren van JCL, Python een volwaardige vervanger van Rexx kan zijn.
  
\end{abstract}
