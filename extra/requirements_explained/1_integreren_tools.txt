1. Moet kunnen integreren met tools van Mainframe

Hier heeft Python in inhaalslag te maken en ik heb het idee dat dat ongoing is, maar nog wel een weg te gaan heeft.
Op sommig gebied is Python al volledig bij, bijvoorbeeld werken met RACF (niet alleen met SEAR, maar simpele RACF commands zijn gewoon TSO commands en Python kan prima TSO commands aftrappen met de Unix 'tsocmd' functie), bijvoorbeeld werken met SDSF want er zijn zover ik weet al diverse Python packages om SDSF dingen te doen (al heb ik er zelf nog geen ervaring mee, dus beetje op glad ijs van mijn kant), bijvoorbeeld werken met z/OS in de zin van data sets alloceren, bekijken, verwijderen, console commands afvuren, jobs submitten enz. doordat de zogenaamde Z Open Automation Utilities (ZOAU) ook een Python package leveren.
Waar Python nog een weg te gaan heeft denk ik (maar ik heb me niet volledig verdiept op het internet) is bijvoorbeeld integratie met 'middleware' tools zoals CICS, DB2, MQ, IMS e.d. Voor sommige functionaliteiten wordt vanuit die 'middleware' producten een alternatieve route geboden, bijvoorbeeld een HTTP interface van CICS om bepaalde administratieve handelingen in CICS te doen (CMCI interface) en Python is dan weer als een vis in het water als het gaat om HTTP calls. Maar niet alle benodigdheden zijn op die manier beschikbaar denk ik.
Dit is een moving target, dus IBM snapt dit ook en komt met meer en meer Python ondersteuning. Daarnaast speelt veel nieuwe ontwikkeling op Z zich af in de USS omgeving van z/OS en staat daarmee automatisch al dichter bij Python. Maar omdat z/OS een bijzonder diverse omgeving is zal het niet meevallen om álles in Python beschikbaar te krijgen.
Je ziet die diversiteit bijvoorbeeld terug in de rollen van verschillende mainframe engineers. Een DevOps is vooral geïnteresseerd in het pipelinen van builds en deploys. Een sysprog is geïnteresseerd in infra-as-code en de langzame beweging naar zero-touch systemen. Een security admin zoekt misschien meer naar selfservice loket functionaliteit en reporting op de RACF database. Enz.enz.enz. Elke discipline is op reis naar een meer en meer gescripte geautomatiseerde omgeving en in elke oplossing is 'lijm' nodig en er is nog geen taal die álles aan elkaar kan lijmen.
